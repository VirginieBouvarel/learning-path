## Appris aujourd'hui

- **Choisir le type d'un champ SQL** : correspondance avec les types TypeScript (ENUM = union type, UUID = branded type, TEXT vs VARCHAR)
- **Pointeurs et clés étrangères** : un pointeur est une référence vers autre chose — la clé étrangère est son implémentation SQL. Le pointeur est toujours sur l'enfant, pas sur le parent.
- **Pourquoi le pointeur est sur l'enfant** : une cellule SQL = une valeur, pas un tableau. L'enfant s'accumule en pointant vers le parent immuable. L'API restitue ensuite la structure imbriquée naturelle attendue côté front.
- **Les 3 types de relations** : one-to-many (clé étrangère sur l'enfant), many-to-many (table de jointure), one-to-one (fusion des tables en pratique)
- **YAGNI appliqué au modèle de données** : ne pas ajouter de colonne si aucune fonctionnalité V1 ne la lit
- **Elephant Carpaccio** : appliqué au périmètre de KataSensei lui-même (V1 réduite, cohérente, livrable)
